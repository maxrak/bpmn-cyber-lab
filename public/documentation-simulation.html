▶️ Simulation - Simulazione Token

Il Simulation permette di eseguire simulazioni del flusso di processo usando token che si muovono attraverso il diagramma BPMN, visualizzando l'esecuzione in tempo reale.

Come funziona la simulazione

  1. Carica un diagramma BPMN
    Usa File → Apri BPMN per caricare il diagramma che vuoi simulare.
  2. Attiva la modalità simulazione
    Clicca sul canvas per attivare la modalità simulazione. Il diagramma mostrerà overlay interattivi sugli elementi.
  3. Avvia il processo
    Clicca sull'Start Event (evento iniziale) per generare il primo token e avviare il processo.
  4. Controlla l'esecuzione
    Usa i pulsanti di controllo nella toolbar:
    • ▶️ Play: Avvia l'esecuzione automatica
    • ⏸️ Pause: Mette in pausa l'esecuzione
    • ⏭️ Step: Esegue un singolo passo
    • 🔄 Reset: Riavvia la simulazione
  5. Monitora il flusso
    Il pannello Simulation Log a sinistra mostra in tempo reale:
    • Token aggiunti/rimossi dagli elementi
    • Stato della simulazione
    • Notifiche ed eventi
    • Timestamp di ogni operazione

Interazione durante la simulazione

🎯 Token sui Task
Quando un token arriva su un task (attività), clicca sull'overlay del task per:
  • ▶️ Completare il task (token prosegue)
  • ⏸️ Mettere in pausa
🔀 Gateway (Punti di decisione)
Nei gateway esclusivi (XOR), clicca sul percorso desiderato per scegliere quale ramo seguire.
📊 Visualizzazione token
I token appaiono come cerchi colorati sugli elementi. Il numero indica quanti token sono presenti sull'elemento.

Log della simulazione

Il pannello log sulla sinistra registra tutti gli eventi della simulazione:

  • 🔵 Token aggiunti: Quando un token arriva su un elemento
  • ➕ Token aumentati: Quando più token si accumulano
  • ⚪ Token rimossi: Quando un token lascia un elemento
  • ▶️ Simulazione avviata/in pausa: Cambio di stato
  • 🏁 Processo completato: Quando tutti i token raggiungono gli end event

Usa il pulsante Pulisci Log per cancellare le voci precedenti.

Controlli di simulazione

▶️ Play
Avvia l'esecuzione automatica del processo. I token si muoveranno automaticamente attraverso il diagramma seguendo il flusso definito. Il sistema monitora in tempo reale il movimento dei token e aggiorna il log ogni 500ms.
⏸️ Pause
Mette in pausa l'esecuzione automatica. Utile per analizzare in dettaglio lo stato corrente del processo o per prendere decisioni manuali ai gateway.
⏭️ Step
Esegue un singolo passo della simulazione. Ideale per un controllo granulare del flusso e per comprendere esattamente cosa succede ad ogni transizione.
🔄 Reset
Riavvia completamente la simulazione, rimuovendo tutti i token e riportando il processo allo stato iniziale. Il log viene mantenuto per riferimento storico.

💡 Suggerimenti per la Simulation

  • Attiva sempre la modalità simulazione: Clicca sul canvas prima di iniziare per vedere gli overlay interattivi.
  • Usa Step per debug: Il controllo passo-passo è ideale per capire esattamente come funziona il tuo processo.
  • Controlla il log: Il pannello log è essenziale per tracciare il flusso e identificare anomalie.
  • Monitora i token bloccati: Se un token rimane fermo su un elemento, potrebbe esserci un problema nel flusso del processo.
  • Simula più istanze: Clicca più volte sullo Start Event per vedere come si comporta il processo con più istanze parallele.
  • Usa Pause per analisi: Metti in pausa quando vuoi esaminare in dettaglio lo stato corrente della simulazione.

❓ FAQ - Simulation

Non vedo i pulsanti di simulazione

Assicurati di aver caricato un diagramma BPMN valido. I controlli di simulazione appaiono solo dopo aver caricato un file.

La simulazione non avanza

Verifica che la modalità simulazione sia attiva (clicca sul canvas). Poi clicca sull'Start Event per generare il primo token.

Come gestire gateway complessi?

Nei gateway esclusivi (XOR), devi scegliere manualmente il percorso cliccando sul ramo desiderato. Nei gateway paralleli (AND), tutti i rami vengono eseguiti automaticamente.

Posso simulare più istanze di processo contemporaneamente?

Sì! Clicca più volte sullo Start Event per generare più token e simulare più istanze di processo che corrono in parallelo.

Il log è troppo lungo

Usa il pulsante Pulisci Log in alto nel pannello per cancellare le voci precedenti e iniziare da zero.

I token si muovono troppo velocemente

Usa il pulsante Pause per fermare l'esecuzione automatica, o usa Step per avanzare un passo alla volta.

Come faccio a sapere se il processo è completato?

Il log mostrerà il messaggio "🏁 Processo completato - nessun token attivo" quando tutti i token hanno raggiunto un End Event e non ci sono più token attivi nel sistema.

Posso salvare lo stato della simulazione?

Attualmente non è possibile salvare lo stato della simulazione. Ogni volta che carichi un diagramma, la simulazione parte da zero. Il log viene mantenuto fino a quando non lo cancelli manualmente o ricarichi la pagina.
