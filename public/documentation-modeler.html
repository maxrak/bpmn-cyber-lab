🎨 Modeler - Editor BPMN

Il Modeler permette di creare e modificare diagrammi BPMN 2.0 con supporto per proprietà di sicurezza informatica personalizzate.

Iniziare

  1. Carica un diagramma esistente
    Usa il menu File → Apri BPMN nella navbar o il pulsante nella toolbar per caricare un file .bpmn esistente.
  2. Crea un nuovo diagramma
    Usa il pannello degli strumenti a sinistra per aggiungere elementi BPMN al canvas:
    • Start Event: Evento di inizio processo
    • Task: Attività/compito da eseguire
    • Gateway: Punti di decisione (XOR, AND, OR)
    • End Event: Evento di fine processo
  3. Collega gli elementi
    Clicca su un elemento e trascina i connettori per creare Sequence Flow tra gli elementi.
  4. Configura le proprietà
    Seleziona un elemento e usa il pannello delle proprietà a destra per:
    • Modificare nome e documentazione
    • Configurare proprietà tecniche (ID, tipo)
    • Aggiungere proprietà cyber personalizzate
  5. Salva il diagramma
    Usa File → Salva BPMN per esportare il diagramma in formato .bpmn

Funzionalità avanzate

🔍 Zoom
Usa i pulsanti + e - nella toolbar o la rotella del mouse per zoomare il canvas.
↶ Undo/Redo
Annulla e ripristina modifiche usando i pulsanti nella toolbar o le scorciatoie Ctrl+Z / Ctrl+Y.
🔒 Proprietà Cyber
Il pannello proprietà permette di aggiungere attributi di sicurezza informatica personalizzati agli elementi del processo per analisi di rischio.

Scorciatoie da tastiera

  • Space + drag: Sposta il canvas
  • Canc: Elimina elemento selezionato
  • Ctrl+C / Ctrl+V: Copia/Incolla
  • Ctrl+A: Seleziona tutto

🔒 Proprietà Cybersecurity

BPMN Cyber Lab estende lo standard BPMN 2.0 con proprietà personalizzate per l'analisi del rischio di sicurezza informatica. Queste proprietà permettono di annotare i task con informazioni critiche per la valutazione della sicurezza del processo.

Come accedere alle proprietà cyber

  1. Seleziona un task
    Clicca su un elemento di tipo Task, User Task o Service Task nel diagramma.
  2. Apri il pannello proprietà
    Il pannello apparirà automaticamente sulla destra. Scorri fino alla sezione "Cybersecurity".
  3. Compila i campi
    Inserisci i valori appropriati per ciascuna proprietà cyber disponibile.

Proprietà disponibili

🎯 STRIDE
Categorie di minacce secondo il modello Microsoft STRIDE:
  • S - Spoofing (falsificazione identità)
  • T - Tampering (manomissione dati)
  • R - Repudiation (ripudio azioni)
  • I - Information Disclosure (divulgazione informazioni)
  • D - Denial of Service (negazione del servizio)
  • E - Elevation of Privilege (elevazione privilegi)
Esempio: S,T,E indica che il task è vulnerabile a Spoofing, Tampering ed Elevation of Privilege.
⚔️ MITRE ATT&CK
Identificatori delle tecniche di attacco dal framework MITRE ATT&CK. Specifica quali tecniche di attacco potrebbero essere applicate a questo task.
Esempio: T1078 (Valid Accounts) o T1078,T1552 per tecniche multiple.
Consulta il catalogo MITRE: attack.mitre.org
📊 Risk Score
Punteggio numerico del rischio calcolato come Probabilità × Impatto.
Convenzione suggerita:
  • 🟢 0-30: Rischio basso
  • 🟡 31-60: Rischio medio
  • 🟠 61-90: Rischio alto
  • 🔴 91-100: Rischio critico
Esempio: 12 per rischio basso, 85 per rischio alto.
I task vengono colorati automaticamente in base al risk score!
📝 Audit Level
Livello di dettaglio richiesto per l'audit e il logging delle operazioni:
  • None: Nessun requisito di audit
  • Basic: Logging base delle operazioni principali
  • Detailed: Logging dettagliato di tutte le operazioni e accessi
Esempio: Task critici come "Provisioning account" dovrebbero avere livello Detailed.
⚠️ Single Point of Failure (SPOF)
Indica se il task rappresenta un singolo punto di fallimento critico per il sistema:
  • No: Task non critico o con ridondanza
  • Yes: Task critico senza alternative - il suo fallimento blocca l'intero processo
Esempio: Un servizio di provisioning centralizzato senza backup dovrebbe essere marcato come Yes.

Esempio pratico

Consideriamo un task di "Provisioning Account" in un processo IAM:

Provisioning account (Service Task)
STRIDE: T,E
MITRE ATT&CK: T1078,T1552
Risk Score: 20 🟢
Audit Level: Detailed
SPOF: Yes ⚠️

Interpretazione: Questo task è vulnerabile a Tampering ed Elevation of Privilege, potrebbe subire attacchi di tipo Valid Accounts (T1078) o Unsecured Credentials (T1552), ha un rischio basso (20) ma richiede audit dettagliato ed è un singolo punto di fallimento critico.

Colorazione automatica

Il sistema applica automaticamente una colorazione ai task in base al Risk Score per facilitare l'identificazione visuale dei task più critici:

  • 🟢 Verde (0-30): Rischio basso
  • 🟡 Giallo (31-60): Rischio medio
  • 🟠 Arancione (61-90): Rischio alto
  • 🔴 Rosso (91-100): Rischio critico

La colorazione viene applicata automaticamente al caricamento del diagramma e quando si modifica il Risk Score nel pannello proprietà.

📄 Formato File BPMN

I diagrammi vengono salvati in formato BPMN 2.0 XML, lo standard internazionale per la modellazione di processi. I file hanno estensione .bpmn e sono compatibili con altri strumenti BPMN come Camunda Modeler.

💾 Compatibilità
Puoi importare diagrammi creati con altri editor BPMN 2.0 e esportare i tuoi diagrammi per usarli in altre piattaforme.

💡 Suggerimenti per il Modeler

  • Salva frequentemente: Usa regolarmente la funzione di salvataggio per non perdere il lavoro.
  • Nomi significativi: Usa nomi descrittivi per gli elementi per rendere il processo più comprensibile.
  • Annota i task critici: Usa le proprietà cyber per documentare le vulnerabilità e i rischi di ogni task. Questo facilita l'analisi della sicurezza del processo.
  • Carica esempi annotati: Apri i file come iam_lifecycle_risk_annotated.bpmn per vedere esempi di annotazioni cyber complete.
  • Sfrutta la colorazione automatica: Il Risk Score colora automaticamente i task - usa questo per identificare rapidamente le aree critiche del processo.
  • Test con Simulation: Dopo aver creato un diagramma nel Modeler, testalo sempre con Simulation per verificare il flusso.
  • Organizza il layout: Mantieni il diagramma pulito e ordinato per facilitare la comprensione del processo.

❓ FAQ - Modeler

Come aggiungo nuovi elementi al diagramma?

Usa la palette degli strumenti sul lato sinistro del canvas. Clicca sull'elemento desiderato e poi clicca sul canvas dove vuoi posizionarlo.

Non vedo il pannello delle proprietà

Assicurati di aver selezionato un elemento cliccandoci sopra. Il pannello delle proprietà appare sulla destra quando un elemento è selezionato.

Come collego due elementi?

Clicca sull'elemento di partenza, poi clicca e trascina uno dei cerchi blu che appaiono verso l'elemento di destinazione.

Posso annullare un'operazione?

Sì! Usa Ctrl+Z per annullare e Ctrl+Y per ripristinare, oppure i pulsanti nella toolbar.

Come esporto il diagramma in un'immagine?

Attualmente l'export è disponibile solo in formato BPMN XML. Per ottenere un'immagine, puoi usare lo strumento di cattura schermo del tuo sistema operativo.

Non vedo la sezione "Cybersecurity" nel pannello proprietà

La sezione Cybersecurity appare solo quando selezioni un Task, User Task o Service Task. Altri elementi come Start Event, End Event o Gateway non hanno proprietà cyber.

Cosa sono i codici MITRE ATT&CK?

MITRE ATT&CK è un framework che cataloga le tecniche di attacco informatico. I codici come T1078 identificano tecniche specifiche. Puoi consultare il catalogo completo su attack.mitre.org per trovare i codici più appropriati per il tuo caso.

Come calcolo il Risk Score?

Il Risk Score è tipicamente calcolato moltiplicando la Probabilità (quanto è probabile che la minaccia si verifichi) per l'Impatto (quanto danno causerebbe). Ad esempio: Probabilità 4 (su scala 1-10) × Impatto 5 (su scala 1-10) = Risk Score 20. Puoi usare il tuo metodo di valutazione preferito.

Le proprietà cyber vengono salvate nel file BPMN?

Sì! Le proprietà cyber vengono salvate come estensioni Camunda nel file BPMN XML. Quando riapri il file, tutte le proprietà saranno preservate insieme alla colorazione automatica basata sul Risk Score.

Posso usare i file con proprietà cyber in altri strumenti?

I file BPMN con proprietà cyber sono compatibili con altri editor BPMN 2.0, ma le estensioni personalizzate (STRIDE, MITRE, Risk Score, ecc.) potrebbero non essere visualizzate in altri strumenti. Le proprietà rimangono comunque nel file XML e saranno visibili quando lo riapri in BPMN Cyber Lab.
